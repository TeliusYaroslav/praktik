Сама задача заключалась в написании файла который будет обрабатываать файлы с расширением .ini искать в них ошибки , сравнивать значения и все это логгировать 
для этого я использовал три библиотеки 
re для вывода текста в столбик в логгировании 
logging для логгирования в отдельный файл (логгирования то есть вывода ошибок предупреждений и т.д)
configParser для собственно работы с файлом .ini для извлечения из него данных и отображения их как section 
Настройки логирования (функция setup_logging)
Эта функция создаёт файл rechspez_protokol2.txt, куда записываются все сообщения о работе программы. Она задаёт формат сообщений: [OK] для успеха, [WARN] для предупреждений и [NOK] для ошибок.

Загрузка файла настроек (функция load_ini_file)
Эта функция открывает .ini файл, читает его содержимое и проверяет, есть ли в нём данные. Если файл пустой или повреждён, она пишет об этом ошибку.

Проверка "BildRefRechner" (функция check_bildrefrechner)
Эта функция ищет ключи, начинающиеся на BildRefRechner (например, BildRefRechner1, BildRefRechner2). Она проверяет, чтобы номера шли подряд без пропусков. Если обнаруживает пропуски, записывает ошибку.

Проверка уникальности "KFSchluessel" (функция check_unique_kf_schluessel)
Эта функция ищет ключи KFSchluessel в секции и проверяет, чтобы их значения не повторялись. Если есть дубли, она пишет об этом ошибку.

Проверка последовательности "KFSchluessel" (функция check_sequential_kf_schluessel)
Здесь проверяется, чтобы ключи KFSchluessel шли по порядку, например: KFSchluessel1, KFSchluessel2, и так далее. Если порядок нарушен, функция сообщает об этом.

Основная функция (функция checkfunk)
Эта функция запускает весь процесс. Она загружает файл, проверяет его содержимое и выполняет описанные выше проверки для секций, названия которых начинаются с "RECHNER". Остальные секции просто пропускаются.
